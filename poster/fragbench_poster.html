FragBench: Cross-Session Attacks Hidden in Benign-Looking Fragments

Astha Mehta  ·  Niruthiha Selvanayagam  ·  Hengxu Li  ·  Raymond Lee  ·  Olivia McGoffin  ·  My (Isabella) Luong  ·  Arthur Collé  ·  David Williams-King  ·  Linh Le

Problem / Motivation

LLMs are dual-use — the same capabilities that help developers also enable data exfiltration, ransomware, and exploitation tooling.

Real incidents: Anthropic's GTG-1002 (AI-orchestrated espionage, ~30 targets) and GTG-2002 (AI-calibrated ransom, 17 orgs).

Key attack pattern: attackers split a malicious goal into benign sub-tasks across separate sessions with no shared context.

Safety classifiers see one prompt at a time → miss the malicious composition. Cross-session fragmentation is unstudied.

🔑 Contributions

  • Malicious dataset: 24 real-world cyber campaigns, 22 MITRE ATT&CK techniques
  • Benign dataset: Synthetic matched cover sessions with MCP execution traces
  • FragBench-Attack: RL-style rewriter + MCP execution validator
  • FragBench-Defense: User-level graph detector (GNN + classical baselines)
  • Tiered HuggingFace release (public + gated)

Attack

Stage 1 — Adversarial rewriting: Safety judge returns ALLOW / REFUSE / AMBIGUOUS. RL rewriter loops with reward +1/−1/+0.3.

Three phases: (1) Bootstrap — rule-based warm start (keyword replace, legitimacy prefix, structural reframe); (2) In-context learned — Sonnet 4.6 applies prior successes to new fragments; (3) Creative — free-form rewriting conditioned on exemplars.

Stage 2 — MCP execution: Actor model runs fragments via tools; checker verifies kill chain was realised. 4 failure modes patched by hand.

Defense

Build typed interaction graph: 121-dim feature vector per tool-use event node.

5 edge types: DATA_FLOW, TEMPORAL, SHARED_SESSION, SHARED_RESOURCE, ARGUMENT_SIMILARITY. Weighted union-find recovers attack chains.

Classifiers evaluated: GCN, GraphSAGE, GAT, GIN (GNNs) + SVM, MLP, GBT (classical baselines).

Dataset Construction

Pipeline:

  1. Seed prompt from real incident report
  2. Claude Opus splits into 4–16 fragments
  3. Variable substitution across 100 variations
  4. State linkage via produces / consumes
  5. 6 style disguises applied per fragment

Scale: 649,076 MCP events, 93,280 tool calls.

Benign data: Synthetic cover sessions (7–14 frags) matching style & length distribution. Themes: sysadmin, IT support, compliance, documentation, project management.

Discussion

Key takeaways:

  1. Cross-session fragmentation defeats single-turn safety classifiers by construction — each fragment passes individually; malicious intent is compositional.
  2. The malicious signal is recoverable only via user-level graph modeling — not isolated prompts.
  3. Defense requires modeling cross-session interaction graphs, not just individual requests.
  4. All artifacts (generator, rewriter, sandbox harness, detector) released open-source with tiered HuggingFace access.
Limitations & Ethics

Benign dataset generated by Claude Sonnet 4.5 — may be statistically distinguishable on unmeasured axes. Dataset is a synthetic recreation of publicised attacks only. Most sensitive rewriter prompts are withheld behind HuggingFace gated access to limit misuse.
